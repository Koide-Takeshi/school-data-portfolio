# Webサイト形態素解析・集計ツール (scraping_analysis)

学校の授業課題として制作したWebスクレイピング + 日本語テキスト分析システムです。

## 概要

指定したWebサイトからテキストデータを収集し、Janomeで形態素解析を行い、名詞の出現頻度を集計・出力します。辞書のカスタマイズやNGワード設定にも対応しています。

## 担当範囲とブラッシュアップ

グループ制作では、主にJanomeを使った形態素解析と単語集計部分を担当しました。制作後のブラッシュアップでは、ポートフォリオとして説明しやすく、実行時にも壊れにくい形に整えています。

| 課題 | 改善内容 | 実務的に良くなった点 |
|------|----------|----------------------|
| 実行場所によって辞書・出力ファイルを見つけられない | スクリプト自身の場所を基準にパスを解決 | bat実行でもPython直接実行でも動かしやすい |
| 辞書追加ファイルとNGワードファイルの指定が混ざりやすい | `add_dicword.txt` と `ng_word.txt` の用途を分離 | 設定ミスによる解析結果のブレを防ぎやすい |
| 通信失敗時の原因が分かりにくい | HTTPタイムアウトとステータス確認を追加 | ネットワーク/サイト側の問題を切り分けやすい |
| Excel出力が結果確認用として最低限だった | 見出し行・列幅・固定表示を追加 | 出力結果をそのまま成果物として見せやすい |

## 実行方法

```
sites_data_count_project/sites_data_count.bat をダブルクリック
```

または Python 環境で:

```bash
cd sites_data_count_project
python 01_src/sites_data_count.py
```

## 主な機能

- 複数URLのテキストデータを一括取得
- Janomeカスタム辞書による専門用語対応
- NGワードリストによる不要語除外
- 名詞の出現頻度をSQLiteに保存 → Excelで出力

# 不動産物件管理・家賃推定システム (rent_prediction_system)

学校の授業課題として制作した不動産物件データベース管理システムです。機械学習（XGBoost）による家賃推定機能を搭載しています。

## 概要

SQLiteデータベースで物件情報を管理し、学生用・スタッフ用のメニュー画面から物件検索・登録・編集を行えます。XGBoostモデルを用いて物件条件から家賃を推定するAI機能も実装しています。

## 担当範囲とブラッシュアップ

グループ制作では、主に学生用・職員用メニューなどのGUI部分を担当しました。制作後のブラッシュアップでは、画面遷移とエラー表示を中心に、実務で使う時に困りやすい点を整理しています。

| 課題 | 改善内容 | 実務的に良くなった点 |
|------|----------|----------------------|
| 実行場所によって子画面を起動できないことがある | スクリプト自身の場所を基準に起動パスを解決 | bat実行・Python直接実行のどちらでも安定しやすい |
| 推定モデルファイルが無いと起動時にクラッシュする | GUI上にモデル読込エラーを表示 | 利用者が原因を把握しやすい |
| 職員用メニューの月額費用推定が停止表示のままだった | 実装済みの推定画面へ接続 | メニューから機能をたどれる |
| 検索結果テーブルの列設定が重複していた | 列定義をリスト化してループで生成 | 列の追加・修正がしやすい |

## 技術スタック

| カテゴリ | 技術 |
|---------|------|
| 言語 | Python 3 |
| 機械学習 | XGBoost（家賃推定モデル） |
| データベース | SQLite3 |
| GUI | Tkinter（標準ライブラリ） |
| データ処理 | pandas / joblib |

## フォルダ構成

```
rent_prediction_system/
├── stu_menu.py                # 学生用メニュー（メインUI）
├── stuff_menu.py              # スタッフ用メニュー
├── ai_estim_2.py              # AI家賃推定モジュール
├── bukken_load_3.py           # 物件データ読込み
├── search_1.py                # 検索機能
├── requirements.txt           # 依存ライブラリ
├── xgb_model.pkl              # 学習済みXGBoostモデル
├── features.pkl               # 特徴量定義ファイル
└── Rent.db                    # SQLite物件データベース
```

## 実行方法

```
# 学生用メニュー
stu_menu.bat をダブルクリック

# スタッフ用メニュー
stuff_menu.bat をダブルクリック
```

または Python 環境で:

```bash
pip install -r requirements.txt
python stu_menu.py    # 学生用
python stuff_menu.py  # スタッフ用
```

## 主な機能

- 物件情報のCRUD操作（SQLiteデータベース）
- 条件指定による物件検索
- XGBoostを用いた家賃自動推定
- 学生用・スタッフ用で権限分けされたUIメニュー
